## 一、基础信息收集

### （一）域名搜集

#### 根域名获取

- **工商信息查询**：爱企查、企查查、小蓝本等平台获取公司相关根域名

- **ICP备案查询**：[工信部备案查询](https://beiancx.miit.gov.cn/#/Integrated/index)

- **ICP备案号反查**：将网站ICP备案号直接在 FOFA 中查询

- **域名历史Whois**：[站长之家历史查询](https://whois.chinaz.com/history)

- **SSL证书获取**：通过SSL证书信息获取根域名

- **注册人反查**：[站长之家注册人反查](https://whois.chinaz.com/reverse/register)，输入公司名称查询

- **域名历史记录**：利用 DomainTools、SpyOnWeb 等工具获取域名历史 Whois 记录，查看所有权变更和 DNS 解析历史

#### 子域名收集

- **空间搜索引擎**：ZoomEye、Shodan 等，从网络空间维度发现子域名

- **OneForAll**：[GitHub](https://github.com/shmilylty/OneForAll/releases/tag/v0.4.5)，支持多种数据源，灵活配置参数

- **Subdomainsbrute**：多线程扫描，优化字典和线程数提升效果

- **SubFinder + KSubdomain + HttpX**：SubFinder 快速发现，KSubdomain 深度挖掘，HttpX 验证存活

- **灯塔（ARL）**：资产侦察灯塔系统

- **子域名挖掘机**：[Gitee](https://gitee.com/yemurplayer)

- **SSL证书获取**：[myssl.com](https://myssl.com/)

- **FOFA查询**：`domain="baidu.com"`

- **Google语法**：`site="baidu.com"`

### （二）IP与CDN绕过

#### 获取真实IP

- **多地Ping**：使用超级 ping、多地 ping 服务检测

- **nslookup**：查询 DNS 记录获取 IP，但部分情况受 CDN 影响，**不能绕过CDN**

- **traceroute**：在 Linux 和 macOS 系统中跟踪数据包路由路径，尝试绕过 CDN

- **邮件头信息**：若目标有邮件服务，分析邮件头获取邮件服务器 IP，可能是真实 IP 或相关 IP

- **微步在线**：将 IP 地址输入微步在线判断是否为 CDN

### （三）空白页面处理

遇到空白页或报错页面时：

- 使用 Google 语法 `site:` 查看是否有路径

- 尝试端口扫描

- 使用 dirsearch 等工具爆破路径

## 二、搜索引擎与网络空间引擎

### （一）Google Hacking

#### 常用语法

- `site="baidu.com"` — 搜索子域名或限定搜索范围

- `site="xxx.com" 身份证` — 限定站点搜索敏感信息

- `filetype=` — 指定文件类型搜索

- `inurl:admin login` — 查找含特定关键词的 URL

- `site:example.com 登录 | 管理员 | 学号 | 奖学金 | 身份证 | 账号 | 密码 | admin | login` — 综合搜索敏感信息

#### 在线工具

- [Google Hacking 语法库](https://ght.se7ensec.cn/#) — 查询 GitHub 安全情报、代码泄露或员工信息

- [Google Dorks 生成器](https://dorks.faisalahmed.me/) — Google Dorks 语法库，用于发现敏感信息和漏洞

### （二）网络空间搜索引擎

常用平台：

```Plain Text
FOFA   Hunter   360Quake   Shodan   ZoomEye   00信安
```

#### FOFA常用语法

- `domain="baidu.com"` — 域名搜索

- `icp="xxxx"` — ICP备案号搜索

- `cert="xxx"` — SSL证书搜索

- `title="xxx公司"` — 标题关键词搜索

- `body="xxx公司"` — 正文关键词搜索（可在静态资源中提取特殊内容）

- `icon_hash="xxxxxxxxxx"` — 图标Hash搜索（FOFA会显示图标hash值）

> 在常规手段用完过后，可通过网络空间引擎补充排查遗漏的资产

#### 常见漏洞特征搜索

- **Spring Actuator**

    - `body="Whitelabel Error Page"`

    - `body=""timestamp":"2025" && body=""status": 404" && port="8081" && country!="CN"`

- **若依 Druid**

    - `body="请通过前端地址访问"`

    - 路径拼接 `/druid/login.html` → 弱口令（`ruoyi/123456`、`ry/123456`）

- **WebLogic**

    - `body="10.4.5 404 Not Found"`

- **Jeecg-Boot**

    - `app="JeecgBoot-企业级低代码平台"`

#### 搜索登录/后台/注册

```Plain Text
domain="xxx.com" && (title="登录" || title="后台" || "管理")
domain="xxx.com" && (body="注册" || body="验证码登录")
```

## 三、网络与系统架构信息

### （一）端口扫描与网络服务

- **Nmap**

    - TCP SYN 扫描（`-sS`）：隐蔽性好

    - 脚本扫描（`-sC`）：获取详细信息

    - 版本探测（`-sV`）：获取服务版本信息

- **Tscanplus**：备选扫描工具

- **网络拓扑**

    - SolarWinds Network Topology Mapper：自动发现设备绘制拓扑图

    - Netdiscover：专注局域网设备发现

> 扫描完成后，直接访问 IP 地址，HTTP 不行则尝试 HTTPS

### （二）操作系统识别

- **Ping TTL值**：Windows TTL≈128，Linux TTL≈64；TTL>100 一般为 Windows，几十一般为 Linux。⚠️ 不准确，且 Ping IPv6 无 TTL 值

- **Nmap**：`nmap -O IP` 获取操作系统指纹（可能被防护机制阻止）

- **大小写敏感性**：Windows 不区分大小写，Linux 区分大小写

- **P0f**：被动嗅探网络流量，分析数据包特征识别操作系统类型、版本及网络配置

### （三）中间件与容器识别

- **F12 响应头 Server 字段**：查看 "Server" 字段，如 "Apache"、"Nginx" 等

- **在线工具**：WhatWeb、Wappalyzer

- **错误页面分析**：不同中间件错误页面有特定信息，如 Tomcat 含版本信息

### （四）网站服务与脚本类型

#### 判断方法

- **文件扩展名**：`.js` → JavaScript，`.php` → PHP，`.jsp` → Java，`.asp/.aspx` → ASP.NET

- **Server 字段**：Apache、Nginx 等

- **Cookie 值分析**

- **在线工具**：WhatWeb

#### 中间件与脚本类型关联

- **IIS** → `.asp` / `.aspx`

- **Tomcat、Resin、JBoss** → `.jsp`

- **Nginx** → PHP、Python（Flask、Django 等）

### （五）数据库类型

#### 前后端对应关系

- **ASP / .NET** → SQL Server、Access

- **PHP** → MySQL、PostgreSQL

- **Java** → Oracle、MySQL

#### 常见数据库端口

- **Oracle**：1521

- **MySQL**：3306

- **SQL Server**：1433

- **PostgreSQL**：5432

- **MongoDB**：27017

#### 判断方法

- 使用 WhatWeb

- SQL 注入判断

### （六）CMS 识别

- **常见 CMS**：WordPress、Joomla、Drupal、Dedecms（织梦）、Discuz、PHPcms 等

- **检测工具**：CMSeeK、joomscan、WPScan、TPscan 等

## 四、企业与社会工程信息

### （一）集团与子公司

> 企业 SRC 不只是代表该公司，而是代表整个集团，少部分 SRC 会收取整个集团的漏洞

- 全资子公司一般都会收，控股 > 50% 也可能会收

- **自动化收集工具**：[ENScan_GO](https://github.com/wgpsec/ENScan_GO)

### （二）企业架构与人员数据

- **组织画像**：查询子公司、合作伙伴、供应商信息，关注行业动态、并购和项目合作，挖掘潜在攻击入口

- **人员信息**：通过社交媒体、GitHub 泄露代码或历史密码库搜集

- **Maltego**：整合关联信息构建关系网络

## 五、业务与应用信息

### （一）敏感目录与文件

- **常见敏感文件**：`robots.txt`、`crossdomain.xml`、`sitemap.xml`、后台目录、网站安装目录等

- **目录扫描工具**：feroxbuster、dirb、dirsearch、dirmap、DirBuster、御剑

    > 根据目标选择合适字典，调整扫描线程数和超时时间

### （二）漏洞扫描与 WAF 识别

- **漏洞扫描**：OpenVAS（开源、社区活跃、规则库更新快）

- **WAF 识别**

    - wafw00f：[GitHub](https://github.com/EnableSecurity/wafw00f) / [Kali](https://www.kali.org/tools/wafw00f/)

    - Nmap 脚本：`nmap -sV --script=http-waf-detect <目标IP>`

    - 手动测试：发送攻击请求观察目标响应判断 WAF 及防护强度

### （三）业务逻辑与权限

- **登录入口**：探测后台地址、API 接口及默认凭据，分析 URL 结构和资源文件找隐藏入口

- **权限提升**：分析用户角色，利用 Burp Suite 抓包修改参数测试越权漏洞，结合业务逻辑找提升机会

### （四）历史漏洞关联

- 利用 CVE Details、SecurityFocus 等数据库查询同类系统漏洞

- 关注安全厂商和社区预警修复建议

## 六、信息泄露

### （一）源码泄露

- **GitHub 泄露**：开发人员上传代码未删除重要信息

- **.git 泄露**：版本控制自动部署时配置不当，导致 `.git` 目录部署到线上

    - **防护**：部署时删除 `.git` 文件夹，Nginx 配置中屏蔽 `.git` 目录访问，定期检查清理敏感文件和目录

- **在线搜索**

    - [searchcode.com](https://searchcode.com/)

    - [github.com](https://github.com/)

### （二）代码与配置文件泄露

#### 版本控制文件泄露（.git、.svn 等）

- **下载完整 .git 目录**

    - 使用 git-dumper、GitHack

    - 若工具无法使用，可用 wget 递归下载

- **分析提交历史**

    - 查看日志，关注作者、时间及提交说明

    - `git log --stat` — 显示文件变更统计

- **恢复历史版本**

    - `git reset --hard <commit_hash>` — 回滚查看被修改的关键文件

- **挖掘敏感信息**

    - 直接搜索代码中的敏感信息

    - 查看 API 接口，寻找后台管理界面

### （三）敏感数据直接暴露

- **API 接口未授权访问**：水平越权

- **通信数据监听**：Wireshark 拦截未加密的 HTTP 流量

- **访问日志信息**：未授权的日志文件泄露用户输入内容

### （四）调试与报错泄露

- **触发调试异常**：输入错误参数查看报错，可能包含绝对路径、错误信息等

- **前端代码泄露路径**：分析 JavaScript、CSS 文件，提取编码的后台路径或内网服务地址

### （五）备份与缓存泄露

- **备份文件猜测下载**：通过字典爆破备份文件路径，利用常见命名规则

- **缓存数据读取**：访问 CDN 缓存节点或浏览器缓存，获取过期未清理的数据
